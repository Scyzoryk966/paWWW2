Umieszcanie skryptu w pliku HTML

Skrypt w pliku HTML umieszcamy w taki sposob:

<script type="text/javascript">
       tutaj kod skryptu
</script>

w sekcji body lub head.

Instrukcja document.write

Żródło: webmade.org

Nasz pierwszy skrypt będzie wypisywał na ekranie tekst. Posłużymy się do tego instrukcją document.write. Document jest to obiekt JavaScript, który reprezentuje aktualnie wyświetlaną stronę, natomiast write - to jego metoda, czyli funkcja wykonująca określone działania na obiekcie - w tym wypadku wypisuje tekst. Nasz tekst umieszczamy w nawiasach, jako argumenty wywołania metody. Ogólnie postępujemy następująco: obiekt.metoda( argumenty metody ) Zatem nasz standardowy program "Hello World" będzie wyglądał następująco:

document.write("Witaj świecie!")

Okna dialogowe

Żródło: kursjs.pl

Z jednej strony okienka takie są łatwe we wdrożeniu i skuteczne (bo gdy się pojawią, nie pozwalają kliknąć niczego innego na stronie), z drugiej strony bardzo ładnie odciągają uwagę odwiedzającego od zawartości strony. Nie odradzam ich używania. Po prostu czasami trzeba się zastanowić, czy nie ma lepszej metody, by poinformować użytkownika o danym zdarzeniu. Przykładowo zamiast pokazywać alerta z informacją o źle wpisanym mailu, lepiej jest wyświetlić obok takiego pola czerwony znaczek czy coś symbolizującego, że ów pole jest źle wypełnione. Użytkowanie strony będzie wtedy o wiele przyjemniejsze.

alert('treść komunikatu');	//okienko alert
prompt('treść komunikatu');    //okienko prompt
confirm('treść komunikatu');    //okienko confirm

Zmienne

Żródło: robert.type.pl

W języku JavaScript można wykonywać operacje nie tylko na liczbach, ale także na innych typach wartości. Pełna lista tych typów jest następująca (w nawiasach są nazwy angielskie):

  1. liczby (Number);
  2. ciągi znaków (String);
  3. wartości logiczne (Boolean);
  4. obiekty (Object);
  5. tablice (Array);

oraz dwa typy/wartości specjalne: Null i Undefined. Wykonamy nastepujący program:

<script language="javascript">
var imie = "Damian"
var nazwisko = "Szefler"
var wiek = "23"
var napis = "Nazywasz się " + imie + " " + nazwisko + " i masz " + wiek + " lat.";
document.write(napis);
</script>

Żródło: javascript.xn--rzeniczak-sbc.pl

W dwóch powyższych linijkach kodu użyliśmy operatora + łączenia stringów. Dzięki niemu możemy wypisywać złożone zdania, zmieniające się w zależności od zmiennych. Jeżeli przy łączeniu różnych typów zmiennych występuje string i operator łączenia +, pozostałe zmienne są przekształcane również na string. Zestawienie operatorów używanych w JavaScript:

Instrukcja warunkowa

Żródło: robert.type.pl

Do stworzenia prostego skryptu wystarczy użyć kilku (lub więcej) instrukcji, które zostaną wykonane sekwencyjnie (jedna po drugiej), tak jak były zapisane. W praktyce jednak takie proste skrypty są rzadkością - zdecydowanie częściej w skrypcie trzeba wykonać (lub nie) dany fragment kodu w zależności od tego czy określony warunek będzie spełniony (lub też odwrotnie). Do tego celu właśnie służą instrukcje warunkowe, które omówię w tej części kursu. Instrukcje warunkowe służą do decydowania który fragment kodu powinien zostać wykonany w zależności od spełnienia określonych warunków; inaczej mówiąc służą do rozgałęzienia ścieżki wykonywania skryptu. Wykonamy nastepujace skrypty:

<script type="text/javascript">
  var a = prompt("Podaj imie...", "...");
    if (a!=null){
        document.write("Witaj "+a+" !");
        }
    else {
        document.write("Nie podałeś imienia!");
        }
</script>
<script type="text/javascript">
	var a = prompt("Podaj A...", "...");
	var b = prompt("Podaj B...", "...");
	var c = prompt("Podaj C...", "...");
	var delta;
		delta = (b * b) - 4 * a * c;
			if(delta>0){
			alert("Wynik to: " +delta);
			}
			else{
			alert("Jestes w dupie, delta jest ujemna");
			}
</script>

Pętla For

Stworzenie skryptu który ma po kolei wykonać kilka operacji jest proste, jak to przedstawiłem w poprzednich lekcjach kursu. Jednak nierzadko zdarza się iż pewną operację lub sekwencję operacji warto byłoby wykonać wielokrotnie, chociażby z tego względu aby zaoszczędzić sobie pisania. Owszem można skopiować dany fragment skryptu tyle razy ile potrzeba, ale do jest kiepskie rozwiązanie. Po pierwsze tak przygotowany skrypt jest trudny w utrzymaniu - wyobraź sobie tylko że skopiowałeś(aś) go 20 razy, i musisz w tych 20 miejscach nanieść jakąś poprawkę. Po drugie takie podejście nie zawsze jest możliwe - czasami trzeba powtarzać daną operację, za każdym razem sprawdzając czy określony warunek jest spełniony (np. wczytywać od użytkownika kolejne liczby aż do momentu aż wpisze on liczbę zero). Do rozwiązania tego typu problemów stosuje się pętle. Użycie pętli spowoduje że określony fragment skryptu będzie wykonany określoną z góry ilość razy, lub też do momentu aż podany warunek zostanie spełniony.Wykonamy nastepujacy skrypt:

<script type="text/javascript">
    for (var i=1;i<=50;i++) {
        if(i%2 !=0){
                document.write("<br>");
        }
        else{
            document.write(+i);
        continue
        }
    }
</script>

Program wypisującydaną ilosć kratek:

<script type= "text/JavaScript">
    var e=prompt("Podaj liczbe f " , "" );
    var f;
    for (f=1;f<=e;f++)
    {
        document.write("#");
    }
    document.write("<br>");
</script>

Program wypisujący trójkąt prostokatny o danej podstawie:

<script type="text/javascript">
    var a=prompt("Podaj liczbe a " , "" );
    var c;
    var d;
    var e=1;
    for (c=1;c<=a;c++)
        {
            for (d=1;d<=e;d++)
            {
            document.write("#")
            }
        e++;
        document.write("<br>")
        }
    document.write("<br> <br> <br>");
</script>

Program wypisujący Prostokat o podanych bokach:

<script type= "text/JavaScript">
    var a=prompt("Podaj liczbe a " , "" );
    var b=prompt("Podaj liczbe b " , "" );
    var c;
    var d;
        for (c=1;c<=a;c++)
        {
            for (d=1;d<=b;d++)
            {
                document.write("#")
            }
            document.write("<br>")
        }
    document.write("<br> <br> <br>");
</script>

Program wypisujący tabliczke mnoczenia danych liczb:

<script type= "text/JavaScript">
    var a=prompt("Podaj liczbe a " , "" );
    var b=prompt("Podaj liczbe b " , "" );
    var i;
    var j;
    for(i=a;i<=b;i++) {
        for (j=a;j<=b;j++)
        {
        document.write(j*i+" ")
        }

    }
</script>

Pętla while

Żródło: webmaster.helion.pl

Pętlami nazywamy konstrukcje języka, które pozwalają na wielokrotne wykonywanie powtarzających się instrukcji. Przykładowo, jeśli trzeba 10 razy wyświetlić na ekranie pewien napis, to można wykorzystać w tym celu 10 instrukcji document.write("napis"), jednak zdecydowanie lepiej i prościej użyć jednej z pętli. Program będzie wyglądał następująco:

<script type="text/javascript">
  var a=1;
  while(a<=20){
    document.write("Nie lubie poniedziłków<br>");
    a++
  }
</script>

Ten sprypt wypisuje wszystkie liczby pomiedzy A i B lub sprzawdza czy sa równe:

<script type="text/javascript">
  var a = prompt("Podaj mniejsza.");
  var b = prompt("Podaj wieksza...");
  if(a<b){
    while(a<b-1){
      a++;
      document.write(+a+"<br>");
      }
  }
  else if(a>b){
    while(a-1>b){
    a--;
    document.write(+a+"<br>");
    }
  }
  else if(a==b){
  document.write("A jest równe B");
  }
</script>

Pętla do...while

Żródło: lomilowka.pl

Podobnym rodzajem pętli jest pętla typu do ... while. Zasadniczą różnicą między tym typem a poprzednim jest to, że kod który ma być powtarzany zostanie wykonany przed sprawdzeniem wyrażenia. Ogólna postać tego typu pętli jest następująca:

do {
    ...fragment kodu który będzie powtarzany...
} while (wyrażenie)

Wykonamy nastepujace zadanie: 

<script type="text/javascript">
                var x = 1;
                var haslo;
                do {
                        haslo = prompt("Podaj hasło...", "...");
                        if (haslo == "barbarossa") {
                                document.write("Złamałeś haslo!<br>");
                                break
                        } else {
                                document.write("złe hasło<br>");
                                x++
                                continue
                        }
                }
                while (x < 4);
                if (x == 4) {
                        document.write("Przekroczyles ilosc prob zalogowania<br>");
                }
</script>

Funkcja

Żródło: zsoitmilowka.pl

Gdy mąż słyszy od żony idź do sklepu, kup ziemniaki, wówczas zaczyna wykonywać pewien zbiór czynności. Za każdym razem, gdy usłyszy idź do sklepu, kup ziemniaki, wykona ten sam zbiór czynności (heh - do czasu...). W javascript takie czynności ujęte w grupę to właśnie funkcje. Funkcja jest wywoływana przez inną część skryptu (w realnym życiu przez żonę), a w momencie jej wywołania zostaje wykonywany kod w niej zawarty. Ogólny wygląd funkcji ma następującą postać:

function nazwa_funkcji() {
    ...kod funkcji
}
//lub
nazwa_funkcji = function() {
    ...kod funkcji anonimowej
}

Wykonamy nastepujace zadanie:

<script type="text/javascript">
    function pisz() {
        alert("To jest tekst zawarty w funkcji.");
    }

    pisz(); //odpalamy powyższą funkcję
    }
</script>

Obiekty

Żródło: robert.type.pl

Pewnie słyszeliście już nazwę "programowanie obiektowe"? Jeżeli jeszcze nie programowaliście obiektowo, to nie ma się co martwić - JavaScript właściwie cała opiera się na programowaniu obiektowym ;). W waszych skryptach nieraz i nie dwa odwołujecie się do obiektów. Na przykład window, document, button itp. to typowe obiekty, które posiadają swoje metody i właściwości :). Oczywiście jak i inne języki, tak i Javascript umożliwia tworzenie własnych obiektów. Początkowo może się wam wydawać, ze jest to nie za bardzo użyteczne (w końcu można większość rzeczy zrobić przy pomocy funkcji itp), ale gdy zaznajomicie się z obiektami na pewno zmienicie zdanie. Przede wszystkim obiekty dają nam ochronę danych. Dziwnie brzmi prawda? Na stronie nie raz tworzyłeś button. Każdy z nich ma np. metodę onclick, właściwość value itp. Nie ma żadnego konfliktu w skrypcie, mimo tego, że obiekty te mają metody i właściwości takie same jak ich "bracia". Poza tym programowanie obiektowe jest najbardziej bliskie rzeczywistości. Zapraszam do lektury.

Aby utworzyć nowy pojedynczy obiekt możemy skorzystamy z poniższej konstrukcji:

var obiekt_1 = {
    imie: "Marcin",
    wzrost: 184,
    wypisz : function() {
        alert(this.imie)
    }
}

Tworzenie obiektu za pomocą konstruktora:

function _KlasaObiektu(_width,_height) {
    this.width = _width;
    this.height = _height;
    this.wypisz = function() {
        alert(this.width + 'x' + this.height)
    }
}
